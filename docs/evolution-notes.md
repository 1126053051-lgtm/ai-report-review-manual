# 修订方向笔记（非版本，仅方向探索）

基于 v3.8 实践后整理的几条转向思路，不代表任何正式版本。

## 定位转向：法律条文 → 编译器指令

| 维度 | v3.8 风格 | 转向方向 |
|------|-----------|---------|
| 定位 | 像法律条文，假设AI会自觉遵守 | 像编译器指令，只定义输入输出边界 |
| 执行 | 依赖AI"自检"和"宣誓" | 依赖外部工具和结构化验证 |
| 语言 | "严禁"、"必须"、"深刻反思" | "若无URL则删除"、"置信度<0.8则标记存疑" |
| 目标 | 产出完美的报告 | 产出可追溯、可证伪的中间过程 |

## 核心本体论：从"生成"转向"映射"

- **反创造力原则**：AI在此语境下不是创作者，而是数据转录员。禁止"润色"、"补全"或"推断"。
- **无源即删**：无法在搜索结果中直接定位的实体，必须在输出前物理删除，而非"待核实"。

## 修订动力学：对抗性补丁机制

- **证据驱动迭代**：每次修订必须由确凿的幻觉证据触发，而非理论上的完善。
- **防御性编程思维**：将AI视为具有"编造冲动"的漏洞代码，每一条新规则都是一个安全补丁。

## 执行纪律：过程可见性

- **链式思维强制**：必须输出"检索-比对-裁决"中间日志。无日志的输出默认无效。
- **信源等级森严化**：T3引用必须显性标注局限性。

## 裁决与熔断：零容忍红线

- **确定性分级**：D3内容必须拦截，不得进入最终交付物。
- **自动熔断**：检测到"据行业普遍观察"等模糊填充词，触发流程重启。
